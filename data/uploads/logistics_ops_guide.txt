1. 개요
-------
본 문서는 물류 운영 시스템의 핵심 데이터 흐름과 테이블 간 관계를 정의한다.
모든 입고·출고·재고·배송 이력은 아래 명시된 테이블 구조를 기반으로 적재되며,
담당자는 쿼리 작성 시 FK(외래키) 조인 조건을 반드시 준수해야 한다.


2. 마스터 데이터 구조
---------------------

[MST_PRODUCT] — 상품 마스터
  - PK: PRODUCT_NO
  - 상품의 기본 속성(무게, 부피, 보관 온도 조건)을 관리한다.
  - TEMP_COND 컬럼은 ROOM / COOL / FROZEN 세 가지 값만 허용된다.
    냉장(COOL) 및 냉동(FROZEN) 상품은 반드시 MST_WAREHOUSE.TEMP_ZONE이
    일치하는 창고에만 배치되어야 한다.
  - SUPPLIER_CD 컬럼은 MST_SUPPLIER.SUPPLIER_CD를 참조한다 (FK).
  - STATUS = 'INACTIVE'인 상품은 신규 입고 및 출고 지시 대상에서 제외한다.

[MST_SUPPLIER] — 공급업체 마스터
  - PK: SUPPLIER_CD
  - LEAD_TIME_DAY는 발주 후 입고까지의 평균 소요 일수이며,
    재발주점(MST_PRODUCT.REORDER_POINT_QTY) 계산 시 활용된다.
  - RATING = 'C'인 공급업체는 분기 성과 리뷰 대상이며,
    LEAD_TIME_DAY가 7일 초과 시 자동으로 이슈 로그가 생성된다.

[MST_WAREHOUSE] — 창고/물류센터 마스터
  - PK: WAREHOUSE_CD
  - CAPACITY_CBM은 창고의 최대 보관 용량으로, 실시간 재고
    (FACT_INVENTORY_SNAPSHOT.STOCK_QTY × MST_PRODUCT.VOLUME_CBM)의
    합계가 CAPACITY_CBM의 90%를 초과하면 포화 경보가 발령된다.
  - TEMP_ZONE = 'MIXED'인 창고는 ROOM/COOL/FROZEN 모두 보관 가능하나
    구역 분리 운영이 필수이다.

[MST_CARRIER] — 운송사 마스터
  - PK: CARRIER_CD
  - SLA_DAY는 배송 약속 기준 일수이며, FACT_DELIVERY_TRACKING.DELAY_DAY
    계산의 기준이 된다. (DELAY_DAY = 실제 배송일 - PROMISED_DT)
  - RATING = 'C' 이하 운송사는 신규 경로(MST_ROUTE) 배정 금지 원칙이 적용된다.

[MST_ROUTE] — 배송 경로 마스터
  - PK: ROUTE_CD
  - ORIGIN_WAREHOUSE_CD → MST_WAREHOUSE.WAREHOUSE_CD (FK)
  - CARRIER_CD → MST_CARRIER.CARRIER_CD (FK)
  - 경로별 담당 운송사와 예상 소요일(EST_TRANSIT_DAY)을 사전 등록하며,
    FACT_OUTBOUND에서 ROUTE_CD 배정 시 이 테이블을 기준으로 SLA를 판단한다.


3. 팩트 데이터 흐름
-------------------

[FACT_INBOUND] — 입고 실적
  - PK: INBOUND_NO
  - FK: PRODUCT_NO → MST_PRODUCT.PRODUCT_NO
  - FK: SUPPLIER_CD → MST_SUPPLIER.SUPPLIER_CD
  - FK: WAREHOUSE_CD → MST_WAREHOUSE.WAREHOUSE_CD
  - 입고 검수 후 STATUS를 RECEIVED → INSPECTED → PUTAWAY 순으로 갱신한다.
  - DEFECT_QTY > 0인 경우, 불량률(DEFECT_QTY / INBOUND_QTY × 100)이
    3%를 초과하면 해당 SUPPLIER_CD의 품질 이슈로 등록된다.
  - PUTAWAY 완료 시 FACT_INVENTORY_SNAPSHOT의 INBOUND_QTY에 반영된다.

[FACT_OUTBOUND] — 출고 실적
  - PK: OUTBOUND_NO
  - FK: PRODUCT_NO → MST_PRODUCT.PRODUCT_NO
  - FK: WAREHOUSE_CD → MST_WAREHOUSE.WAREHOUSE_CD
  - FK: CARRIER_CD → MST_CARRIER.CARRIER_CD
  - FK: ROUTE_CD → MST_ROUTE.ROUTE_CD
  - 출고 지시 생성 시 FACT_INVENTORY_SNAPSHOT.AVAILABLE_QTY 기준으로
    재고 가용성을 확인한다. AVAILABLE_QTY < OUTBOUND_QTY이면 부분 출고 처리.
  - PROMISED_DT = OUTBOUND_DT + MST_ROUTE.EST_TRANSIT_DAY 로 자동 산정.
  - STATUS 흐름: PICKED → PACKED → SHIPPED → DELIVERED

[FACT_INVENTORY_SNAPSHOT] — 재고 스냅샷
  - 복합 PK: (SNAPSHOT_DT, PRODUCT_NO, WAREHOUSE_CD)
  - FK: PRODUCT_NO → MST_PRODUCT.PRODUCT_NO
  - FK: WAREHOUSE_CD → MST_WAREHOUSE.WAREHOUSE_CD
  - 매일 23:59 배치로 당일 FACT_INBOUND.INBOUND_QTY와
    FACT_OUTBOUND.OUTBOUND_QTY를 집계하여 STOCK_QTY를 갱신한다.
    공식: STOCK_QTY(D) = STOCK_QTY(D-1) + INBOUND_QTY(D) - OUTBOUND_QTY(D)
  - AVAILABLE_QTY = STOCK_QTY - RESERVED_QTY
  - STOCKOUT_YN = 'Y' 조건: AVAILABLE_QTY <= 0
  - 품절(STOCKOUT_YN = 'Y') 발생 시 SCM팀에 자동 알림 발송.

[FACT_DELIVERY_TRACKING] — 배송 추적
  - PK: TRACKING_NO (운송장 번호)
  - FK: OUTBOUND_NO → FACT_OUTBOUND.OUTBOUND_NO
  - FK: CARRIER_CD → MST_CARRIER.CARRIER_CD
  - FK: ROUTE_CD → MST_ROUTE.ROUTE_CD
  - DELAY_DAY = ACTUAL_DELIVERY_DT - FACT_OUTBOUND.PROMISED_DT
    (양수: 지연, 음수: 조기 도착)
  - DELIVERY_STATUS = 'FAILED' 또는 'RETURNED'인 경우,
    FAIL_REASON 컬럼에 사유를 기록하고 재배송 프로세스를 트리거한다.
  - DELAY_DAY >= 2인 건은 운송사 SLA 위반으로 집계되며,
    MST_CARRIER.RATING 재평가에 반영된다.


4. 핵심 KPI 및 집계 로직
-------------------------

[적시 배송률 (OTD, On-Time Delivery)]
  - 산식: DELIVERED 건 중 DELAY_DAY <= 0인 비율
  - 참조 테이블: FACT_DELIVERY_TRACKING JOIN FACT_OUTBOUND (OUTBOUND_NO)
                 JOIN MST_CARRIER (CARRIER_CD)
  - 목표: 95% 이상 / 월별 MST_CARRIER.RATING 재산정 기준

[입고 불량률]
  - 산식: SUM(DEFECT_QTY) / SUM(INBOUND_QTY) × 100
  - 참조 테이블: FACT_INBOUND JOIN MST_SUPPLIER (SUPPLIER_CD)
                 JOIN MST_PRODUCT (PRODUCT_NO)
  - 목표: 2% 이하 / 초과 시 공급업체 개선 협의 트리거

[재고 회전율]
  - 산식: 월 출고 수량(SUM FACT_OUTBOUND.OUTBOUND_QTY) /
          월 평균 재고(AVG FACT_INVENTORY_SNAPSHOT.STOCK_QTY)
  - 참조 테이블: FACT_OUTBOUND, FACT_INVENTORY_SNAPSHOT
                 모두 PRODUCT_NO, WAREHOUSE_CD 기준 JOIN
  - 목표: 카테고리별 상이 (일반 상품 월 4회 이상)

[창고 가동률]
  - 산식: SUM(STOCK_QTY × VOLUME_CBM) / CAPACITY_CBM × 100
  - 참조 테이블: FACT_INVENTORY_SNAPSHOT JOIN MST_PRODUCT (PRODUCT_NO)
                 JOIN MST_WAREHOUSE (WAREHOUSE_CD)
  - 목표: 70~85% 유지 (90% 초과 시 포화 경보)


5. 주요 이슈 및 운영 현황 (2025년 3월 기준)
--------------------------------------------

[이슈 1] 냉장 상품 TEMP_COND='COOL' 배송 지연
  - FACT_DELIVERY_TRACKING 분석 결과, COOL 조건 상품의 DELAY_DAY 평균 2.3일
  - 원인: MST_ROUTE에서 COOL 전용 운송사(CARRIER_TYPE='TRUCK')가 수도권에 집중
  - 조치: 부산·광주 권역 MST_ROUTE에 냉장 CARRIER_CD 추가 배정 필요

[이슈 2] A공급업체 SUPPLIER_CD='SUP001' 리드타임 급증
  - 최근 30일 FACT_INBOUND 기준 평균 LEAD_TIME_DAY = 11일 (기준: 7일)
  - INBOUND_QTY 대비 DEFECT_QTY 비율 4.2% → 불량률 기준 초과
  - MST_SUPPLIER.RATING 재평가 예정 (A → B 조정 검토)

[이슈 3] WH003 창고 포화 임박
  - FACT_INVENTORY_SNAPSHOT 최신 데이터 기준
    WH003(MST_WAREHOUSE.WAREHOUSE_CD='WH003') 가동률 88.7%
  - FROZEN 구역 재고 과적 상태: TEMP_COND='FROZEN' 상품 STOCK_QTY 급증
  - 조치 방향: 일부 재고를 WH005(MIXED 구역)로 이동 또는
    FACT_OUTBOUND 출고 우선순위 조정


6. 데이터 품질 규칙
-------------------

- FACT_INBOUND.WAREHOUSE_CD는 MST_WAREHOUSE.STATUS='ACTIVE'인 창고만 허용
- FACT_OUTBOUND.CARRIER_CD는 MST_CARRIER.STATUS='ACTIVE'이고
  해당 ROUTE_CD의 MST_ROUTE.CARRIER_CD와 일치해야 함
- FACT_INVENTORY_SNAPSHOT.STOCK_QTY는 음수 불가
  (음수 발생 시 데이터 오류로 간주, 운영팀 즉시 확인 필요)
- FACT_DELIVERY_TRACKING.ACTUAL_DELIVERY_DT는
  PICKUP_DT보다 이전일 수 없음
